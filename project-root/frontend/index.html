What is MLQ Scheduling?

Multilevel Queue (MLQ) scheduling partitions the ready queue into multiple separate queues. Each queue can use a different local scheduling policy such as RR, FCFS, or SJF. A global priority decides which queue is served first.

  • Higher-priority queue executes before lower-priority queues.
  • Each queue can have independent logic and time quantum.
  • Preemption can occur when a higher-priority queue becomes ready.

Key Formulae

Turnaround Time (TAT) = Completion Time - Arrival Time

Waiting Time (WT) = Turnaround Time - Burst Time

Response Time (RT) = First CPU Start - Arrival Time

The simulator computes these for each process and average values for each algorithm.
